三、关于 Docker Context

1. 什么是 Docker Context

  • Docker Context 是 Docker CLI 用来管理不同 Docker 主机/环境的配置集合。
  • 每个 context 包含以下信息：
  1. Docker daemon 的 连接方式（Unix socket、TCP 地址等）
  2. 用户认证信息
  3. 可选的 CLI 配置（比如 TLS）

换句话说，context 就像一个 预设的目标环境，你切换 context 就相当于切换了 Docker CLI 要操作的主机。

2. 为什么有 context

  • 方便在同一台机器上操作 多个 Docker 实例：

  • 本地原生 Docker Engine

  • Docker Desktop VM

  • 远程服务器上的 Docker

  • 不用每次都手动指定 DOCKER_HOST、TLS 等参数

4. 总结

  • Context = Docker CLI 的目标环境配置
  • 典型用途：
  1. 在本地开发时切换到 Docker Desktop VM
  2. 在生产服务器上操作远程 Docker
